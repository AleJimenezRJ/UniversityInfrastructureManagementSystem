## 1. Sprint Goal

The goal is to provide an even better user-friendly interface for managing these entities in the system.This following standardization will allow for easier management of the entities in the system, as well as better integration with other systems in the future.

---

## 2. Accepted Product Backlog Items

- **PQL-WV-004-008 Language compliance**: Ensuring that the user interface is compliant with the language requirements of the system (SPANISH))
- **PQL-WV-004-007 Standarized errors**: Ensuring that the errors in the user interface are standardized and consistent across the system
- **PQL-WV-004-004 Update Building Window**: Refactor the building window to ensure that it is user-friendly and easy to use
- **PQL-LB-002-003 Pagination in list buildings**: Implementing pagination in the list of buildings to improve performance and usability
- **PQL-WV-004-006 Standarized UI design**: Ensuring that the user interface is standardized and consistent across the system
- **PQL-AU-012-001 Authorization to manage buildings in the user interface**: Implementing authorization to manage buildings in the user interface

---

## 3. Rejected Product Backlog Items

- All product backlog items were accepted, as they were all completed and meet the acceptance criteria defined in the sprint planning.

---

## 4. Feedback Received

- Countries should be added on a database level to allow for future expansion
- It's required authorization in the backend to ensure that only authorized users can perform certain actions

---

## 5. Actions and Next Steps

- More frequent consultation and validation with the product owner and technical advisors  
- Improve coordination with teams that depend on our work  
- Implement a more robust authorization system in the backend to ensure security and proper access control
- Start working on the next sprint early to ensure that we can deliver the next set of features on time
